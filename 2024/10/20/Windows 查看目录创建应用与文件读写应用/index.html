Windows 查看目录创建应用与文件读写应用

要查询文件或目录的创建者或者创建应用，以下是几种常用的工具和方法：

1. Windows 资源管理器

  • 右键属性：通过右键点击文件或文件夹，选择“属性”，在“安全”选项卡中查看文件的所有者。
  • 详细信息选项卡：有时在“详细信息”选项卡中可以找到文件的相关信息（如创建时间、修改时间等），虽然不直接显示创建应用，但可以根据时间推测。

2. Windows 事件查看器

  • 事件日志可以帮助查看与文件系统操作相关的日志（如创建、修改、删除等）。
    1. 打开 事件查看器（Win + R，输入 eventvwr）。
    2. 导航到 Windows 日志 -> 安全。
    3. 搜索相关的事件 ID：
      • 文件/目录创建事件：通常事件 ID 4656 或 4663（“访问某文件”）。
    4. 查找与文件创建时间一致的日志记录，查看创建进程或用户。

3. Process Monitor (ProcMon) - 推荐

  • Process Monitor 是一个实时监控工具，可以监控文件、进程、注册表等活动，非常适合追踪文件或目录的创建。
    1. 下载并运行 Process Monitor。
    2. 设置过滤器（Filter）以查看 文件系统操作（File System Operation）。
    3. 找到与文件/目录创建时间接近的相关日志，查看哪个进程创建了该文件。

4. PowerShell

  • 通过 PowerShell 脚本可以查看文件的元数据，但一般无法直接获取创建者应用程序。不过可以通过文件的时间信息进行推测。

  • 使用 dir 命令可以查看文件的创建时间：

6. 文件审计功能（File Auditing）

  • 文件审计是 Windows 的一种高级功能，可以用来记录文件或目录的访问和操作（如创建、删除、修改）。
    1. 打开文件/目录的“属性”窗口，导航到 安全 选项卡，点击 高级。
    2. 选择 审计 选项卡，然后点击 添加。
    3. 设置要监控的用户或组，以及要监控的操作（如创建文件或文件夹）。
    4. 事件会被记录在 事件查看器 中的“安全”日志。

7. 第三方工具

  • Everything：一个快速的文件搜索工具，可以按文件创建时间、修改时间进行搜索，方便找到文件并推测创建时间及应用。
  • LockHunter：这个工具可以用来查找文件正在被哪个进程使用，从而推断出哪个应用正在使用或创建该文件。

8. Linux/MacOS 方法

在 Linux 和 macOS 下，常用的方法包括：

  • stat 命令：显示文件的详细信息，包括创建时间、修改时间、访问时间。
  • lsof 命令：查看哪个进程在使用某个文件。
  • inotifywait：实时监控目录内的文件变化，查看文件的创建者应用程序。

通过这些方法和工具，你可以有效地追踪文件或目录的创建者以及相关应用程序的操作。
